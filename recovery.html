Help users recognize, diagnose, and recover from errors: "Error messages should be expressed in plain language (no codes), precisely indicate the problem, and constructively suggest a solution."




The 'send me stuff' form that was mentions under error prevention is validated when you click the 'submit' button. If the user forgot a required field this particular field gets an error message next to it. This makes it easy for the user to notice and correct the mistake. There is however no easy way of clearing the entire form to start over.

Another form that provides validation feedback is the 'class schedule' search form. The error messages are clear and understandable and the message box has a link back to the search page, which makes the recovery easier. In over half of the searches I tried out, I received an error message saying that the subject is not offered in the chosen term. This could have been avoided be making the user choose the term first, and then sort the course list based on the chosen term.

The search function provides a spelling suggestion if a word is misspelled. You do not have to confirm that the suggestion is correct. The results for the suggestion is presented regardless and it saves the user one another click.
